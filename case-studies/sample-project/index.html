Sample Project

Redesigning a team workflow tool to cut task completion time by 40% and boost weekly active usage.

Overview

The client is a B2B SaaS platform used by operations teams to coordinate recurring tasks across departments. Despite strong retention, new users struggled to complete their first workflow without support.

I led the end-to-end design effort to simplify onboarding, clarify task states, and reduce friction in the core daily workflow.

Challenge

User interviews revealed three pain points: unclear task ownership, too many steps to mark work complete, and inconsistent status language across the product. The team needed a redesign that wouldn't disrupt power users while making the product approachable for newcomers.

Process

I started with journey mapping and usability tests on the existing flow. We ran two rounds of moderated testing on low-fidelity prototypes before moving to high-fidelity screens and a clickable prototype for validation.

User journey map
Wireframe exploration

Solution

The final design introduced a unified task card with inline actions, a guided first-run checklist, and a simplified status model with four clear states. Navigation was restructured so the most common actions were always one click away.

Final UI screens

Results

After launch, the team saw measurable improvements in activation and efficiency, with positive feedback from both new and existing customers.

40% Faster task completion
28% Higher activation rate
4.6 Post-launch NPS

Reflection

This project reinforced the value of testing early with real workflows instead of isolated screens. If I were to revisit it, I'd invest more in a scalable component library upfront to speed up engineering handoff.
